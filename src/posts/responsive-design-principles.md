## AI 时代的前端开发范式

AI 的爆发不仅改变了用户界面本身，更深刻改变了前端工程师的工作方式。

## 从编码到策展

过去，前端工程师的核心工作是"写代码"。现在，越来越多的代码由 AI 生成，工程师的角色转变为"策展人"—— 审核、优化和整合 AI 生成的代码。

```typescript
// 传统方式：手写组件
function UserProfile() {
  // ... 几十行状态管理和渲染逻辑
}

// AI 时代：用提示词描述需求，AI 生成代码
// Prompt: "创建一个用户资料卡片组件，包含头像、姓名、简介和编辑按钮"
// AI 生成如下代码，开发者负责审核和调整
```

## 设计到代码的鸿沟在缩小

Figma 插件和 AI 视觉模型可以直接将设计稿转换为代码，但理解设计意图仍然需要人类的判断。好的前端工程师懂得在 AI 生成的代码基础上，注入对用户体验的深刻理解。

## Prompt Engineering 成为新技能

```typescript
// 好的 prompt 能生成更好的组件
const prompt = `创建一个数据表格组件：
- 支持排序和筛选
- 虚拟滚动优化性能
- 可自定义列宽
- 暗色模式适配
- 使用 Tailwind CSS 样式`;
```

AI 不会取代前端工程师，但会用 AI 的前端工程师将会取代不会用 AI 的。
